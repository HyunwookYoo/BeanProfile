## 1. 개요 & 목표

내가 마신 커피 원두(Single Origin / Blend)의 **프로필**과 **시음 경험**을 기록하고,
데이터가 쌓이면 **내 취향 프로필**을 뽑아 보여주는 개인용 앱.

**핵심 플로우 (4단계)**
1. 핸드폰 카메라로 원두 봉투 / 프로필 카드를 촬영한다.
2. 봉투 정보(원산지·농장·컵노트 등)를 **(반자동)** 취합해 기록한다.
3. 실제 시음 경험을 **강도 4축 + 종합 별점 + 코멘트**로 기록한다.
4. 누적 데이터로 **취향 프로필**(선호 강도·원산지·컵노트·가공방식)을 확인한다.

## 3. 데이터 모델

원산지 정보를 별도 엔티티(`OriginComponent`)로 분리한다.
→ Single Origin = 구성 1개, Blend = 구성 2개+ 로 자연스럽게 통합된다.

### `Bean` (원두 = 봉투 하나)
| 필드 | 타입 | 비고 |
|---|---|---|
| id | int (PK) | |
| name | String | 제품명 (예: "Ethiopia Yirgacheffe G1") |
| roaster | String | 로스터리 |
| type | enum `BeanType` | `SingleOrigin` / `Blend` |
| roastLevel | enum `RoastLevel` | Light / LightMedium / Medium / MediumDark / Dark |
| roastDate | Date? | 로스팅 날짜 |
| cupNotes | List\<String> | 컵노트 태그 (자동완성) |
| photoPath | String? | 봉투 사진 로컬 경로 |
| scaScore | double? | 선택 |
| weightGrams | int? | 선택 |
| price | int? | 선택 |
| shop | String? | 구입처, 선택 |
| memo | String? | 선택 |
| createdAt | DateTime | |

### `OriginComponent` (원산지 구성 — Bean 1—N)
| 필드 | 타입 | 비고 |
|---|---|---|
| id | int (PK) | |
| beanId | int (FK) | |
| country | String | 원산지 국가 |
| region | String? | 지역 |
| farm | String? | 농장 / 워싱스테이션 / 협동조합 |
| variety | String? | 품종 |
| process | enum `Process` | Washed / Natural / Honey / Anaerobic / Other |
| altitude | String? | 고도 |
| ratioPercent | int? | 블렌드 비율 % (블렌드일 때만) |

### `Tasting` (시음 기록 — Bean 1—N)
| 필드 | 타입 | 비고 |
|---|---|---|
| id | int (PK) | |
| beanId | int (FK) | |
| date | Date | 시음 날짜 |
| acidity | int 1–5 | 강도 (약→강) |
| sweetness | int 1–5 | 강도 |
| body | int 1–5 | 강도 |
| bitterness | int 1–5 | 강도 |
| overall | int 1–5 | 종합 만족도 (별점) |
| comment | String? | 자유 코멘트 |
| createdAt | DateTime | |

**관계:** `Bean 1—N OriginComponent`, `Bean 1—N Tasting`.

## 4. 화면 구성 & 네비게이션

하단 탭 3개.

### 📚 탭 1 — 원두
- **리스트:** 카드(이름·로스터리·원산지·평균★·썸네일) + 검색 + 정렬(최근/평점/이름), 우하단 `+`
- **원두 추가 플로우:** 봉투 촬영/갤러리 선택 → OCR 리뷰 화면(§5) → 저장 → 상세
- **원두 상세:** 사진 + 프로필(블렌드는 구성 리스트로 표시) + 컵노트 칩 + 평균★·시음 횟수 / 하단에 시음 리스트 + `시음 추가` 버튼 / 편집·삭제
- **시음 기록 화면:** 날짜 · 강도 4축 슬라이더 · 종합 별점 · 코멘트 → 저장

### 📊 탭 2 — 취향
분석 대시보드 (§6).

### ⚙️ 탭 3 — 설정
- **데이터 내보내기 / 가져오기 (JSON)** — 로컬 저장이므로 기기 이전·백업용
- 앱 정보

### UI 목업 & 디자인 방향 (승인 2026-07-14)

- **목업 (5개 화면):** [`docs/mockups/ui-mockup-v1.html`](mockups/ui-mockup-v1.html) · [아티팩트](https://claude.ai/code/artifact/8fb51c60-5550-479a-bdd8-d94ccd093fcc)
- **컨셉:** "커핑 랩 노트" — 시음을 계측하는 정밀 기구 느낌.
- **색:** 웜 오트 바탕(`#ECE6DB`) · 에스프레소 텍스트(`#2B2019`) · **크레마 앰버 단일 강조**(`#B67B2E`, FAB·활성·별점). 흔한 크림+세리프+테라코타 커피앱 클리셰는 회피.
- **타입:** 시스템 산세리프(한글 폴백) + **데이터(로스팅 날짜·고도·비율·평점)는 모노스페이스**로 스펙시트 느낌.
- **핵심 컴포넌트:** 원두 카드 · OCR 2단(자동 필드 + 배정 칩) · 모노 스펙시트 · 강도 5칸 도트 슬라이더 · 종합 별점 · 레이더 + 막대 대시보드.

## 5. 사진 → 정보: OCR 반자동 리뷰

온디바이스 OCR은 글자는 읽지만 "어느 필드인지"는 모른다 → **초안을 깔아주고 사용자가 확정**하는 반자동 방식.

1. 촬영 → 온디바이스 OCR이 **전체 텍스트** 추출 (한글 + 라틴 스크립트 모델)
2. 앱이 **패턴이 명확한 것만 자동 채움**
   - 국가명: 원산지 사전 매칭
   - 로스팅 날짜: 정규식
   - 가공방식 / 로스팅 단계: 키워드 매칭
   - "컵노트 / Notes:" 라벨 뒤 텍스트 → 태그 후보
3. 화면 = 위쪽 **폼 필드**(자동 채운 값은 하이라이트) + 아래쪽 **인식된 텍스트 칩**
4. 사용자가 칩을 탭 → 필드에 배정, 또는 직접 수정 / 입력
5. 저장

> OCR이 실패해도 수동 입력으로 매끄럽게 이어진다.

## 6. 취향 분석 로직 (탭 2)

상단 요약(총 N개 원두 · M회 시음) + 아래 4개 위젯.

| 위젯 | 계산 로직 |
|---|---|
| **① 선호 강도 레이더** | 고평점(종합 ★4~5) 시음들의 평균 산미·단맛·바디·쓴맛 → 4축 레이더 |
| **② 원산지별 평균 평점** | Tasting★을 Bean의 각 구성 `country`에 귀속 → 국가별 평균 (막대) |
| **③ 선호 컵노트 순위** | 고평점 원두의 `cupNotes` 태그 빈도 랭킹 (막대/칩) |
| **④ 가공방식별 평점** | `process`별 평균 종합★ |

**블렌드 귀속 규칙:** 블렌드를 시음하면 그 별점이 구성된 각 국가/가공방식에 데이터 포인트로 들어간다(비율 있으면 가중).

## 7. 기술 스택 & 아키텍처

### 패키지 (Flutter)
| 용도 | 패키지 |
|---|---|
| 로컬 DB | `drift` (SQLite) — 관계형이라 분석 쿼리(조인·group by)에 최적 |
| OCR | `google_mlkit_text_recognition` — 온디바이스·오프라인 (한글+라틴) |
| 카메라/사진 | `image_picker` |
| 차트 | `fl_chart` (레이더 + 막대) |
| 상태관리 | `riverpod` |
| 백업 | `share_plus` + `file_picker` (JSON export/import) |
| 기타 | `intl`(날짜), `path_provider`(사진 저장 경로) |

### 폴더 구조 (feature-first, 가볍게)
```
lib/
  data/        # drift DB, DAO, 테이블 정의
  models/      # Bean, OriginComponent, Tasting
  services/    # ocr_service, backup_service
  features/
    beans/     # 리스트 · 상세 · 추가/편집 · OCR 리뷰
    tasting/   # 시음 기록
    profile/   # 취향 대시보드
    settings/  # 백업/내보내기
  app.dart
```

### 로컬 저장 & 백업
- 모든 데이터는 기기 내 SQLite. 사진은 앱 문서 디렉터리에 저장, 경로만 DB 보관.
- 설정에서 JSON(사진 포함 여부 옵션) 내보내기/가져오기로 기기 이전 대비.
