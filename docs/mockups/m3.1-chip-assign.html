추천 · 안 A

칩 먼저 (칩 탭 → 시트)

스크롤·포커스와 무관, 어디서든 2탭, 모든 필드 대상. 어포던스는 칩 하나뿐이라 깔끔. OCR 멘탈모델(“이 텍스트 뭐지 → 어디 넣지”)과 정확히 일치.

대안 · 안 B

필드 먼저 (칸의 칩 버튼)

“이 칸을 채운다” 사고엔 직관적이나, 칸마다 버튼이 붙어 화면이 복잡해지고 블렌드 구성이 늘면 더 산만.

공통 결정

모든 자유 텍스트 필드 대상화

제품명·로스터리·원산지 국가·지역·컵노트·메모. (지금은 상위 4개만.) 컵노트는 추가(append), 나머지는 교체.

확인 요청

자동값 필드도 시트에?

국가처럼 이미 자동으로 찬 필드도 시트 목록에 넣어 덮어쓰기 허용할지, 아니면 자유 텍스트 칸만 보여줄지.

이 목업은 M3.1 상호작용 확정용입니다. 방향(A/B)·문구·"자동값 필드 포함 여부"를 정하면 설계 스펙 → 구현 계획(SDD)으로 넘어갑니다. 소스는 지금 scratchpad, 확정되면 docs/mockups/로 옮겨 커밋합니다.
