Decision · 차트

의존성 0 — 직접 그린다

막대 3개는 원래 차트가 아니다(track+fill = Container + FractionallySizedBox). 진짜 차트는 레이더 하나뿐이고 4축 고정·값 0~5라 CustomPainter로 충분하다. fl_chart를 들이지 않아 iOS CI 리스크 0이고 목업 색·모노스페이스와 픽셀 단위로 일치한다.

Decision · 집계

SQL이 아니라 순수 Dart

beans.cupNotes는 JSON 문자열 한 칸에 저장돼 SQL로 태그 빈도를 셀 수 없다. 데이터는 수십~수백 행이라 SQL의 성능 이점도 0. 한 번 로드해 순수 함수로 계산하면 4개 위젯이 같은 스냅샷을 공유하고 DB 없이 유닛 테스트된다.

Decision · 블렌드

비율 가중, 원두 단위 all-or-nothing

구성 전부에 비율이 있으면 ratio/100, 하나라도 비면 그 원두는 전 구성 1/n. 부분적으로 채워진 비율을 추측해 메우지 않는다. 싱글 오리진은 항상 w = 1.

Decision · 컵노트

원두 1표

평균★ ≥ 4 인 원두마다 자기 태그에 +1(원두 내 중복 태그는 1회). 막대 숫자가 곧 원두 개수라 직관적이고, 같은 원두를 여러 번 마셨다고 순위가 왜곡되지 않는다.
